Project Description (Ticket Contents)	"This project delivers a pluggable, open-source-first AI microservice platform for a multi-tenant SaaS LMS. It automates the full educational content lifecycle from raw asset ingestion (PDF, PPT, Video, Audio) to publishable interactive micro-learning experiences.
The platform runs entirely on self-hosted, locally-deployed open-source language models (Llama 3, Mistral, Whisper) with no hard dependency on external cloud AI APIs. Output formats are aligned to open LMS standards: HTML5, H5P, and SCORM — renderable on any LMS without proprietary infrastructure.
The platform is structured around four modules:
Module A — Intelligent Document Ingestion (PDF / PPT)
Module B — Automated Assessment Suite (Quiz-Gen)
Module C — Multimedia Intelligence (Video / Audio)
Module D — AI Micro-Learning Studio (HTML5 / H5P / SCORM)
"
Goals & Mid-Point Milestone	"Goals
1. Module A: PDF/PPT parser producing structured JSON, Key Takeaways, Glossary, and narration scripts — all via local LLM.
2. Module B: MCQ, Match-the-Pair, Fill-in-the-Blanks question generator + H5P/SCORM quiz packaging — local LLM as primary engine.
3. Module C: Local Whisper transcription, speaker diarisation, auto-chaptering, H5P Interactive Video with inline knowledge checks.
4. Module D: HTML5/H5P/SCORM micro-lesson generator from any source asset, with tenant branding and HITL review gate.
Goals Achieved By Mid-Point Milestone (Week 6)
- Module A fully operational: ingestion, summarisation, glossary, scripting.
- Module B: MCQ + H5P Question Set packaging working end-to-end.
- Module C: Whisper transcription + VTT output working.
- LLM Gateway (Ollama) live with fallback configuration.
- Async queue (Redis/Celery) operational.
"
Setup/Installation	"Prerequisites: Docker, Docker Compose, Node.js 20+, Python 3.11+
1. Clone the repository and run: docker compose up -d
2. Ollama service starts automatically and pulls Llama 3 8B and Mistral 7B models.
3. Whisper large-v3 is downloaded on first transcription request.
4. Access the API gateway at http://localhost:8000
5. Access the micro-lesson builder UI at http://localhost:3000
Refer to /docs/setup.md in the repository for full environment configuration, tenant setup, and model selection options.
"
Expected Outcome	"1. Fully functional 4-module AI platform deployable on-premise or on any cloud provider.
2. Course creators can ingest a PDF/PPT and produce a structured H5P micro-lesson in under 10 minutes.
3. All AI inference runs on locally-hosted open-source models — no external API dependency by default.
4. Automated quiz generation (MCQ, Fill-in-the-Blanks, Match-the-Pair) packaged as H5P and SCORM packages.
5. Video/audio assets transcribed locally (Whisper), chaptered, and packaged as H5P Interactive Video.
6. All published content meets WCAG 2.1 AA accessibility standards.
7. xAPI tracking for all learner interactions surfaced in the LMS dashboard.
"
Acceptance Criteria	"Module A
✅ PDF and PPT files parsed into structured JSON with headers, body, images, metadata.
✅ Key Takeaways and Glossary generated by local LLM (Llama 3 / Mistral).
✅ Narration script generated from PPT speaker notes.
✅ Processing time: ≤ 30 seconds for a 50-page document.
Module B
✅ MCQ, Match-the-Pair, Fill-in-the-Blanks generated strictly from source material (no hallucinations).
✅ All question types packaged as valid H5P Question Set and SCORM 1.2 packages.
✅ SCORM packages import successfully into Moodle 4.x and Open edX.
✅ LaTeX STEM questions render correctly via MathJax.
Module C
✅ Video/audio transcribed with Whisper (local) — output: VTT + plain-text transcript.
✅ Speaker diarisation labels in transcript.
✅ Auto-generated timestamped chapter markers embedded in HTML5 navigation overlay.
✅ H5P Interactive Video package with inline knowledge checks generated and importable.
Module D
✅ Micro-lesson generated from PDF, PPT, transcript, or free-form input.
✅ Output formats: H5P Course Presentation, HTML5 slide deck, SCORM 1.2 package.
✅ Tenant branding (logo, palette, font) applied automatically.
✅ HITL review gate — no content published without creator approval.
✅ xAPI statements emitted for start, progress, completion, quiz score.
✅ Micro-lesson generation time: ≤ 2 minutes end-to-end.
"
Implementation Details	"Architecture: Microservices (Docker Compose / Kubernetes)
Services:
- Ingestion Service: Python · PyMuPDF · python-pptx
- LLM Gateway: Python · FastAPI · Ollama API (local-first, cloud fallback)
- Assessment Service: Python · h5p-nodejs-library · pipwerks SCORM
- Multimedia Service: Python · Whisper · pyannote-audio
- Micro-Lesson Builder: Node.js · Reveal.js · h5p-cli
- Queue & Worker: Redis · Celery / BullMQ
- Tenant Config Service: PostgreSQL · REST API
AI Model Stack:
- Text Generation: Llama 3 8B (Ollama) — cloud fallback: GPT-4o (optional)
- Transcription: Whisper large-v3 (fully local)
- Embeddings / RAG: nomic-embed-text (Ollama)
- Diagram Generation: Mermaid.js (browser-side, no LLM)
Output Standards: H5P Core 1.24+, SCORM 1.2 / 2004, HTML5, xAPI 1.0, VTT/SRT
"
